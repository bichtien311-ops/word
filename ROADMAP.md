### 1.2. Инфраструктура сборки Word (рабочая, проверена)

- `execution/word_builder.py` — сборка с нуля (`--type methodical_guide`): титул, автооглавление, разделы, введение/заключение, список литературы.
- `execution/word_injector.py` — дозапись/замена в существующий .docx (`append`/`after_heading`/`before_heading`/`replace_section`), режим `--analyze`, `--reformat`.
- `execution/content_processor.py` — парсинг: `clean_notebooklm_output` (чистка ссылок-источников), `fix_numbers` (точки→запятые), `extract_latex_formulas`, `parse_markdown_tables`, `structure_content`.
- `execution/gost_formatter.py` — ГОСТ-стили (TNR 14, поля 30/15/20/20, интервал 1,5, отступ 1,25; заголовки H1 16пт CAPS, таблицы 12пт).
- `directives/` — 4 воркфлоу (создание/вставка/переформатирование/text→word) + новая инструкция `gost_uchebnoe_posobie.md` (структура пособия по ГОСТ).
- Окружение: `.venv` рабочий, `python-docx` установлен (проверено запуском).

## 2. Пробелы и риски

1. **Лекции 6–17 без содержания.** Основной объём генерации. Решается запросами к NotebookLM по каждой лекции с экспертным промптом из «Суть задания».
2. **`план_вопросов.md` под другую тему.** Нужно полностью переписать под 17 лекций по нейтралям.
3. **Метаданные титульного листа не заданы** (вуз, кафедра, автор, дисциплина, город, год, гриф). Нужны от пользователя до финальной сборки — без них титул соберётся с заглушками.
4. **Рендеринг формул.** Сборщик вставляет формулы как линеаризованный текст (LaTeX → plain text через `_latex_to_plain`), а не как нативные формулы Word (OMML). Для первичного черновика приемлемо; для финала — отметить как зону ручной доводки или доработать конвертер.
5. **Единый стиль глубины.** Лекции 1–5 уже подробные; 6–17 нужно довести до того же уровня (5 требований ТЗ), иначе пособие будет неоднородным.
6. **Контрольные вопросы.** В готовых лекциях 1–5 их нет — для уровня методлитературы желательно добавить блок в конце каждой лекции (по ГОСТ-инструкции).

## 3. Пошаговый план (road map)

### Этап 0 — Согласование (нужно от пользователя)
- [ ] Метаданные титула: вуз, кафедра, ФИО автора(ов), дисциплина, город, год.
- [ ] Решение по формулам: оставить линеаризованный текст или дорабатывать нативный рендер.
- [ ] Нужны ли контрольные вопросы и приложения.

### Этап 1 — Стандарт оформления (готово)
- [x] Найдены требования ГОСТ к учебному пособию, сохранены как инструкция `directives/gost_uchebnoe_posobie.md`.

### Этап 2 — План вопросов под нейтрали
- [ ] Переписать `план_вопросов.md`: на каждую из 17 лекций — целевой вопрос к NotebookLM, обёрнутый в CISCO-промпт + 5 требований из «Суть задания» (формулы, алгоритмы, описание диаграмм, нормы ПУЭ/ПТЭ, категории надёжности).

### Этап 3 — Генерация контента лекций 6–17
- [ ] По каждой лекции задать вопрос в NotebookLM, получить ответ с формулами/алгоритмами/нормами.
- [ ] Привести к единому стилю с лекциями 1–5, добавить контрольные вопросы.
- [ ] Сложить в рабочий мастер-файл.

### Этап 4 — Сводный исходник
- [ ] Собрать `data/master_neutrals.md`: введение → модули I–V (лекции 1–17) → заключение → список литературы (ГОСТ Р 7.0.100-2018).

### Этап 5 — Предобработка
- [ ] Прогон через `content_processor.py`: `--clean-nlm`, `fix_numbers`, `--structure`. Проверить распознавание формул и таблиц.

### Этап 6 — Сборка Word
- [ ] `word_builder.py --type methodical_guide` с метаданными → `Результаты/методичка.docx`.

### Этап 7 — Проверка и доводка
- [ ] Открыть .docx, Ctrl+A → F9 (оглавление). Чек-лист ГОСТ: шрифт TNR 14, поля 30/15/20/20, интервал 1,5, отступ 1,25, нумерация страниц, заголовки, таблицы 12пт, формулы, нумерация рисунков/таблиц.

### Этап 8 — Итеративное дополнение
- [ ] По правкам пользователя — дозапись через `word_injector.py` (`after_heading`/`replace_section`, `--apply-gost`), оригинал не перезаписывается.

## 4. Целевая структура пособия (по ГОСТ-инструкции)

1. Титульный лист + оборот (УДК/ББК, аннотация, рецензенты).
2. Оглавление (автособираемое).
3. Список сокращений и обозначений (опц.).
4. Введение.
5. Основная часть:
   - Модуль I. Теоретические основы (лекции 1–3).
   - Модуль II. Изолированная и компенсированная нейтраль (лекции 4–8).
   - Модуль III. Современные режимы заземления 6–35 кВ (лекции 9–11).
   - Модуль IV. Высокие напряжения, неполнофазные и резонансные режимы (лекции 12–15).
   - Модуль V. Релейная защита, безопасность, выбор режима (лекции 16–17).
6. Заключение.
7. Список использованных источников.
8. Приложения (опц.).
