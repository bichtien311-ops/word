# Рабочий процесс: Текст от нейронки → Методичка в Word

## Цель
Принять raw-текст (написанный другой нейронкой), переформатировать его под стандарт методических указаний ВУЗа и вставить в Word-документ.

## Входные данные
Пользователь заполняет файл **`задание.md`** в корне проекта. В нём:
1. **Режим** — что делать:
   - `создать` — новый документ с нуля
   - `добавить` — вставить текст в существующий .docx
   - `заменить` — заменить содержимое конкретной главы
   - `переформатировать` — только ГОСТ-форматирование без изменения контента
2. **Целевой файл** — путь к .docx (для режимов добавить/заменить/переформатировать)
3. **Глава** — название/номер главы (для добавить/заменить)
4. **Позиция** — `после` / `перед` / `в_конец` (для режима добавить)
5. **Текст** — raw-контент от нейронки, вставленный после разделителя `---`

## Пайплайн обработки

### Шаг 1: Прочитать `задание.md`
- Определить режим, целевой файл, главу, позицию
- Извлечь текст после разделителя `---`

### Шаг 2: Обработать текст
Используя `content_processor.py`:
- Очистить от артефактов NotebookLM (`clean_notebooklm_output`)
- Заменить точки на запятые в числах (`fix_numbers`)
- Структурировать в блоки (заголовки, абзацы, формулы, таблицы)

### Шаг 3: Выполнить действие

**Режим «создать»:**
```bash
python execution/word_builder.py \
  --type methodical_guide \
  --input задание.md \
  --output Результаты/методичка.docx \
  --title "..." --university "..." --author "..." --discipline "..." --city "..."
```

**Режим «добавить»:**
```bash
python execution/word_injector.py \
  --target <целевой_файл> \
  --input .tmp/task_content.txt \
  --output Результаты/результат.docx \
  --position after_heading \
  --heading "<глава>" \
  --apply-gost
```

**Режим «заменить»:**
```bash
python execution/word_injector.py \
  --target <целевой_файл> \
  --input .tmp/task_content.txt \
  --output Результаты/результат.docx \
  --position replace_section \
  --heading "<глава>" \
  --apply-gost
```

**Режим «переформатировать»:**
```bash
python execution/word_injector.py \
  --reformat <целевой_файл> \
  --output Результаты/результат.docx
```

### Шаг 4: Проверка
- Открыть полученный .docx в Word
- Ctrl+A → F9 (обновить оглавление, если есть)
- Проверить форматирование по чек-листу из `directives/reformat_document.md`

## Важные правила
1. **Формулы** — берутся 1 в 1 из исходника. Подставляются только числа. Формат: обычная запись + блок LaTeX-кода.
2. **Числа** — все точки в десятичных заменяются на запятые (32.09 → 32,09)
3. **Индексы** — берутся из PDF, т.к. LaTeX из Word невидим для парсера
4. **Язык** — только русский, все обозначения русские
5. **Эталонный файл** — если в имени есть слово «эталон», НЕ трогать, работать только с рабочим файлом
6. **Оригинал НЕ изменяется** — результат всегда сохраняется в отдельный файл

## Структура выходного документа (для режима «создать»)
1. Титульный лист
2. Содержание (автособираемое)
3. Введение
4. Основная часть (главы с нумерацией)
5. Заключение
6. Список использованных источников

## Краевые случаи
- **Нет целевого файла** — если режим «добавить»/«заменить», создать новый
- **Глава не найдена** — сначала запустить `--analyze` для вывода структуры
- **Пустой текст** — предупреждение, ничего не делать
- **Файл с «эталон» в названии** — отказать в модификации
