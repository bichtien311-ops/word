# Создание документа с нуля (Режим 1)

## Цель
Создать полный Word-документ (.docx) по ГОСТ из raw-текста (Markdown, NotebookLM, plain text).

## Входные данные
1. **Raw-текст** — файл `.md` или `.txt` с контентом (заголовки, формулы, таблицы)
2. **Метаданные** — название, вуз, автор, кафедра, дисциплина, город, год
3. **Тип документа** — `methodical_guide`, `lecture`, `study_guide`, `calculation_note`
4. **Источники** (опционально) — список литературы
5. **Введение / Заключение** (опционально) — текст отдельно

## Инструменты
- `execution/content_processor.py` — парсинг и очистка контента
- `execution/gost_formatter.py` — ГОСТ-стили и форматирование
- `execution/word_builder.py` — сборка документа

## Пайплайн

### Шаг 1: Подготовка контента
```bash
python execution/content_processor.py input.md --clean-nlm > cleaned.txt
python execution/content_processor.py cleaned.txt --structure
```
Проверить: формулы LaTeX распознаны, таблицы распарсены, числа → запятые.

### Шаг 2: Создание документа
```bash
python execution/word_builder.py \
  --type methodical_guide \
  --input data/task.md \
  --output .tmp/output/document.docx \
  --title "Электроснабжение жилого микрорайона" \
  --university "КГЭУ" \
  --author "Иванов И.И." \
  --discipline "Электроснабжение" \
  --city "Казань"
```

### Шаг 3: Проверка
- Открыть .docx в Word
- Ctrl+A → F9 (обновить оглавление)
- Проверить: шрифт TNR 14, поля (30/15/20/20), полуторный интервал, нумерация страниц

## Ожидаемый результат
Файл `.docx` с:
- Титульным листом
- Автособираемым оглавлением
- Структурированной основной частью (заголовки, абзацы, формулы, таблицы)
- Списком литературы
- Полным ГОСТ-форматированием

## Краевые случаи
- **Пустой файл** — создаётся документ только с титульным листом
- **Нет LaTeX** — формулы пропускаются, только текст/таблицы
- **Кривой Markdown** — content_processor обрабатывает как обычный текст
- **Файл из NotebookLM** — используйте флаг `--clean-nlm` для удаления ссылок на источники [Source 1] и артефактов
